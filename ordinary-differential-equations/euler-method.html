Mtodo de Euler

O Mtodo de Euler uma tcnica simples de integrao numrica que aproxima a soluo de uma equao diferencial ordinria (EDO) da forma: [ \frac{dy}{dx} = f(x, y), \quad y(x_0) = y_0 ]

Ele calcula a soluo em passos discretos, utilizando a frmula: [ y_{n+1} = y_n + h \cdot f(x_n, y_n) ] onde:

  • ( h ) o tamanho do passo,
  • ( y_{n+1} ) a soluo aproximada no prximo ponto,
  • ( x_n ) o valor atual de ( x ),
  • ( y_n ) o valor atual de ( y ).

Limitaes do Mtodo de Euler

  1. Baixa preciso:

    • O erro acumulado proporcional ao tamanho do passo ( h ).
    • A preciso limitada, pois o mtodo assume que a derivada ( f(x, y) ) constante em cada intervalo.

  2. Estabilidade:

    • Para algumas EDOs, o mtodo pode ser instvel e gerar solues divergentes.

  3. Passos muito pequenos:

    • Para melhorar a preciso, necessrio usar passos menores (( h )), o que aumenta o custo computacional.

Solues Mais Eficientes

Se buscamos maior eficincia e preciso, existem mtodos mais avanados, como:

1. Mtodo de Euler Modificado

Tambm conhecido como Mtodo do Ponto Mdio, ele melhora a preciso ao estimar a derivada no ponto mdio do intervalo: [ y_{n+1} = y_n + h \cdot f\left(x_n + \frac{h}{2}, y_n + \frac{h}{2} \cdot f(x_n, y_n)\right) ] Este mtodo uma boa melhoria em relao ao Mtodo de Euler simples.

2. Mtodo de Runge-Kutta de 4 Ordem (RK4)

Um dos mtodos mais populares, conhecido por sua alta preciso e eficincia. A frmula geral : [ y_{n+1} = y_n + \frac{h}{6} (k_1 + 2k_2 + 2k_3 + k_4) ] onde:

  • ( k_1 = f(x_n, y_n) ),
  • ( k_2 = f\left(x_n + \frac{h}{2}, y_n + \frac{h}{2}k_1\right) ),
  • ( k_3 = f\left(x_n + \frac{h}{2}, y_n + \frac{h}{2}k_2\right) ),
  • ( k_4 = f\left(x_n + h, y_n + h \cdot k_3\right) ).

O RK4 oferece uma soluo precisa com erro proporcional a ( h^4 ), tornando-o amplamente utilizado em problemas prticos.

3. Mtodos Implcitos

Para sistemas rgidos (stiff systems), os mtodos implcitos, como o Backward Euler, so mais estveis. Eles resolvem a equao: [ y_{n+1} = y_n + h \cdot f(x_{n+1}, y_{n+1}) ] Aqui, ( y_{n+1} ) precisa ser resolvido iterativamente.

def euler_method(f, x0, y0, h, n):
    """
    Resolve uma EDO usando o Mtodo de Euler.

    Parmetros:
        f (function): A funo f(x, y) que define dy/dx.
        x0 (float): O valor inicial de x.
        y0 (float): O valor inicial de y.
        h (float): O tamanho do passo.
        n (int): O nmero de passos.

    Retorna:
        list: Lista de pares (x, y) representando a soluo aproximada.
    """
    x = x0
    y = y0
    results = [(x, y)]

    for _ in range(n):
        y = y + h * f(x, y)
        x = x + h
        results.append((x, y))
    
    return results

import numpy as np
import matplotlib.pyplot as plt

# Funo para a EDO
def f(x, y):
    return x + y

# Soluo exata para comparao
def exact_solution(x):
    return 2 * np.exp(x) - x - 1

# Mtodo de Euler
def euler_method(f, x0, y0, h, n):
    x = x0
    y = y0
    results = [(x, y)]

    for _ in range(n):
        y = y + h * f(x, y)
        x = x + h
        results.append((x, y))
    
    return results

# Parmetros
x0 = 0      # Valor inicial de x
y0 = 1      # Valor inicial de y
h = 0.1     # Tamanho do passo
n = 20      # Nmero de passos

# Resolver a EDO com Mtodo de Euler
resultado = euler_method(f, x0, y0, h, n)

# Obter os valores de x e y
x_vals, y_vals = zip(*resultado)

# Gerar valores para a soluo exata
x_exact = np.linspace(x0, x0 + n * h, 1000)
y_exact = exact_solution(x_exact)

# Plotar a soluo aproximada e exata
plt.figure(figsize=(8, 6))
plt.plot(x_vals, y_vals, 'o-', label="Mtodo de Euler (Aproximado)", color="blue")
plt.plot(x_exact, y_exact, '-', label="Soluo Exata", color="red")
plt.title("Soluo da EDO com Mtodo de Euler")
plt.xlabel("x")
plt.ylabel("y")
plt.legend()
plt.grid()
plt.show()

# Mostrar os resultados numricos
print("Resultados do Mtodo de Euler:")
for i, (x, y) in enumerate(resultado):
    print(f"Passo {i}: x = {x:.2f}, y = {y:.4f}")
